Experiment 4 - Infinite Worlds

Description

For each map I used a different action logic. The effects that can be clicked on are different for different maps.

Starry: Clicking on a planet causes the corresponding planet to glow, emitting a more intense light as the number of clicks increases. Planets can be dragged. Clicking any planet four times will turn a planet into a black hole, an irreversible process.

Terrace: Clicking to planting seedlings for fear of terracing.

City: Click on any building to make it taller.

Technical

This time, this integration scheme does not use the “instantiation mode” of p5.js as in the last Ex3, this time, because in order to do switching to prevent different clicks from conflicting, so I made currentWorld to dynamically distribute calls to p3_terrace_*, p3_city_*, p3_starry_*, p3_city_*, p3_starry_*, p3_starry_* and these sets of functions. starry_* sets of functions, so instead of having a new p5 instance for each world. I use a combination of program generation algorithms to create a dynamically evolving virtual landscape based on an example given by my professor. The experiment is centered on a unique hash value generated interactively by the user as a seed to drive the program generation algorithm.
